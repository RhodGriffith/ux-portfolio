Sign in

The sign in page allows a user to gain access to a product by entering their user ID and password.

Overview

Sign in is often the first interaction a user has with our products. This entry point experience is key and sets the tone for their overall experience with the product.

Eliminate non-essential distractions to decrease cognitive load so users are directing to the necessary login flow.

Anatomy of a sign in page

  1. Sign in page title (h1): Located at the top of the sign in flow.
  2. Username input: The username floating input field is required.
  3. Password input: The password floating input field is required.
  4. Keep me signed in: Located under the username and password input fields, the 'Keep me signed in' checkbox is optional. It's designed to give users access to a product without needing to sign in again after a period of inactvity.
  5. Sign in button: The button label should be 'Sign in' to describe the primary call to action. When clicked, the combination of username and password is validated and routes the user to the Multi-Factor Authentication (MFA) page.
  6. Trouble signing in title (h2): Subtitle above the username and password reset links.
  7. Forgor username or password links: Takes users to pages where they can recover their username and/or password.
  8. Help and support: If users are experiencing technical issues signing in, this 'Help and Support' link directs users to the main website where they can contact Customer Support.

Behaviour

Single sign-on (SSO)

SSO enables users to log into multiple UK products through one single authentication rather than using unique usernames and passwords for each product.

Errors and validation

If a user enters an incorrect username or password, a generic error message is will be displayed. This protects valid usernames from being exposed and helps keep products secure.

Multi-factor authentication (MFA)

Multi-Factor Authentication (MFA) provides additional security by requiring a user to complete an additional step to verify their identity. This includes a verification code which is sent via SMS or email.

Design and layout

Placing the sign in form in the center of the screen creates a simple entry point without any distractions. The sign in form remaines fixed width in the centre of the pageuntil the @s breakpoint (500px) where it changes to a fluid layout to fill the width of the screen

By navigating to a sign in page, users have already shown intent to sign in. Introducing additional content that may increase cognitive load at this point is not recommended, the focus should be on the primary goal in hand.

Stacking the interface components vertically reduces eye movement and makes the sign in form easier to read.

Accessibility

Ensure that users can tab through the sign in form and navigate the page using only a keyboard.
