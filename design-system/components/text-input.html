Standard text input

Use the text input component when you need to let users enter information that is no longer than one line.

Placeholders

Example of placeholder hint text inside the input field. Use the u-vh attribute to convey this information to screen reader users as well.

Design guidelines

When to use the standard text input

The standard text input is the default version and should be used in instances when there's a large number of form fields â€“ on a product search page for example.

The bolder text outside of the input area will help users pick out information on the page and will eradicate inconsistencies in labelling when used in conjunction with other form elements.

When to use the floating label text input

Text inputs with floating labels can be used in instances when there's only one or two form fields for a user to complete â€“ on a sign in page for example.

Always place the label above the input field

Floating text labels to the left of input fields disrupts the natural reading pattern and makes it harder for the end user to find the beginning of the next line.

Aligning labels to the left of input fields can cause problems with layout. They don't work well with long labels and there's no room to provide a description if needed.

Accessibility

Don't replace a label with placeholder text

While placeholder text provides valuable guidance for many users, placeholder text is not a replacement for labels. Assistive technologies, such as screen readers, do not treat placeholder text as labels. Placeholder text is not broadly supported across assistive technologies and not displayed in some older web browsers.

Form fields should have visible and persistent labels. Without labels, users cannot check their work before submitting a form and it can also put a strain on short-term memory if validation errors occur.
