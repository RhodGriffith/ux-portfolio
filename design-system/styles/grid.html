Grid

Grids are an essential part of design. They provide solid foundations to divide space, structure content and help communicate hierarchy. Use the Neptune grid system to lay out content on your product's pages.

Creating layouts

Grid columns need to be wrapped using the .grid class.

Each column within .grid requires the class .grid__col and another class to specify the number of columns at the required breakpoint. For example, .col-6@m will provide a single column which is 6 columns wide at 768px and greater. If the screen width is below 768px, the layout will reduce to a single column fill 100% of the content.

A column width can also be set between two breakpoints using the class structure .col-6@m@xl. This will provide a 6 column wide column between 768px and 1280px. Outside of these widths the column will fill 100% of the content.

In the following example, we have a three column layout between 1280px and 1600px, two column layout between 768px and 1279px and a single column below 768px.

Grid modifiers

Push and pull

.push-1@m and .pull-1@m moves the .grid__col the specified number of columns (at the specified breakpoints) either left or right. This is ideal for creating sidebars for secondary content.

Flexbox

.grid--flex .grid__col--flex provides an efficient way to create layouts, align and distribute space among items in a container. There are more flexbox modifiers which can provide the conditions required at specific breakpoints:

  • .grid--center
  • .grid--between
  • .grid--vertical-center
  • .u-flex-shrink@m
  • .u-flex-no-shrink@m
  • .u-flex-grow@m
  • .u-flex-no-grow@m
  • .u-flex-no-basis@m
