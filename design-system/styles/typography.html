Typography

Typography is an essential part of Neptune and brings consistency across all of our products. Good typographic principles establishes visual hierarchy and helps maximise legibility of content and data.

Breakpoint font sizes

Font sizes can be triggered at certain breakpoints. Just suffix any font size class with @{breakpoint}. For example u-fs-xl@xl will change the font size when the browser width is between 1280-1599px.

Open in new window

Example text

<p class="u-fs-l@s u-fs-l@m u-fs-l@l u-fs-xl@xl u-fs-xxl@xxl">Example text</p>

Font weight

You can override Neptune default font weights by using the following classes:

  • u-fw-400 for regular text
  • u-fw-600 for semi-bold text
  • u-fw-700 for bold text

Lists

Lists help the reader identify key points in the text by breaking content into concise and manageable chunks.

Bulleted list

Open in new window
  • Full name
  • National Insurance Number
  • Passport number
<ul class="list"> <li class="list__item">Full name</li> <li class="list__item">National Insurance Number</li> <li class="list__item">Passport number</li> </ul>

Numbered list

Numbered lists should be used for instances in which the items must occur in a specific order, such as steps in a procedure.

Open in new window
  1. Fasten your seatbelt
  2. Start the engine
  3. Look around and drive
<ol class="list"> <li class="list__item">Fasten your seatbelt</li> <li class="list__item">Start the engine</li> <li class="list__item">Look around and drive</li> </ol>

Bare list

Bare lists should be used to display a list of navigational links

Open in new window
  1. Contact us
  2. Help and support
  3. Feedback
<ul class="list list--bare"> <li class="list__item"><a href="#">Contact us</a></li> <li class="list__item"><a href="#">Help and support</a></li> <li class="list__item"><a href="#">Feedback</a></li> </ul>

Design guidelines

Only use Neptune font sizes

Only choose fonts from Neptune's defined sizes. Using custom font sizes will disrupt the hierarchy and balance of typography in our products.

Deviating from default font weights

Sometimes the weight options of Neptune typography components arenâ€™t suitable for specific scenarios. In these situations, itâ€™s OK to deviate from the default weights by specifying a font weight from our list.

Don't use a font weight lighter than Source Sans Pro Regular (400) as it's difficult to read.

Don't change Neptune font colours

Don't override default font colours for typography. For example, red should be used for error messages rather than headlines.

Only use Neptune line heights

Don't override Neptune's defined line heights. Optimal line height is between 120 â€“ 150% of the font size. Using too little or too much leading makes content difficult to read.

Line length

Shorter lines are more comfortable to read than longer lines as the distance your eyes have to travel to the beginning of the next line is reduced.

Line lengths for body text should be between 60 to 90 characters.

Left alight blocks of text

With centre aligned text, users have to work harder to find where each line begins to continue reading.

Don't use capitals for blocks of text

All caps reduces the shape contrast for each word making paragraphs or large blocks of text difficult to read. All caps are fine in contexts that donâ€™t involve much reading, such as sub-headings, acronyms, and abbreviations.

Establish typographic hierarchy

Hierarchy is a way of organising type to establish an order of importance within content and data. This enablers the reader to easily find what they are looking for and navigate content on the page.

Use a combination of size, weight and colour to help establish typographic hierarchy.

Accessibility

Heading ranks

Nest headings in order, for example, <h1>, <h2>, <h3>.

Skipping heading levels should be avoided (e.g. <h1> directly to <h3>) from an accessibility point of view. People using screen readers often rely on navigating by heading so if the structure isn't hierarchical they may not understand the relationship.

Use semantic markup

Use semantic elements to define different parts of a web page.

<article>
<aside>
<details>
<figcaption>
<figure>
<footer>
<header>
<main>
<mark>
<nav>
<section>
<summary>
<time>

Don't justify text

The problem with fully justified text is the kerning, which adds white space between words and letters. This makes the text harder to read, especially for those with cognitive disabilities or dyslexia.
